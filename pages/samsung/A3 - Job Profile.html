Key Objectives
1. Control and manage device by understanding the device and implementing the driver.
2. Contribute to enhanced quality of product through testing and optimizing the device.

Key Tasks
1. [Device analysis] Understand a part of source code that has been basically materialized and understand which parts need to be modified(porting). Understand the operating system driver framework that needs to be modified.
2. [Designing/implementing device driver] Enable(porting) driver to operate within the system with minimal modification.
3. [Optimization of device driver] Review and modify parts that need to be optimized with the range of what has already been modified. Conduct profiling by using profiling tools and methods.
4. [Device driver testing] Conduct testing by using the written test document and understand the result. Add or change some items of test document through approval.

Required Technical Skills
Skill Title
Definition
Understanding the device 
Understand the relevant device through data sheet.
Understanding the circuit related to device
Understand the circuit related to device and identify problems and improvement points.
Understanding OS driver framework
Understand the API and resource management provided by Kernel.
Function materialization
Select operation system device driver framework in order to materialize the device's function and write it with appropriate language.
Materializing non-functional elements
Materialize non-functional elements(performance, power consumption, reliability, product life and etc.) by considering device's characteristics(timing and other physical characteristics such as device's life)
Optimization plan
Identify elements that need to be optimized and define metrics that will be quantified.
Conducting profiling
Develop and introduce profiling tools and use the result to analyze.
Understand the relevant of improvement items
Understand and determine the relationship(tradeoff, inclusive and exclusive) between improvement items and other functional & non-functional elements.
Functional/non-functional testing
Develop plan on functional/non-functional testing of driver and conduct testing.
Driver life and reliability test
Develop plan on the driver life and reliability test conduct testing.
